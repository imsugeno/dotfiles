# Interview Slot Finder

Googleカレンダーの空き状況から、面談やミーティングの候補日程を抽出し、相手に共有しやすいフォーマットで出力するスキル。

## ワークフロー

### Step 1: 条件の確認

ユーザーの依頼から以下のパラメータを特定する。明示されていないものはデフォルト値を使い、確認はまとめて1回で済ませる。

| パラメータ | 説明 | デフォルト |
|---|---|---|
| 対象曜日 | 候補を出す曜日（例: 月・水） | 平日すべて |
| 時間帯 | 候補の下限〜上限（例: 11:00-19:00） | 10:00-18:00 |
| 枠の長さ | 面談に必要な時間（例: 1時間） | 60分 |
| バッファ | 前の予定との最低間隔（例: 15分） | 15分 |
| 候補数 | 提示する日数の上限 | 5つ |
| 検索開始日 | いつから探すか | 翌営業日 |
| タイムゾーン | カレンダーのタイムゾーン | Asia/Tokyo |
| カレンダーID | 対象カレンダー | primary |

### Step 2: カレンダーの空き取得

対象の各日について、以下の2つのAPI呼び出しをセットで行う。

1. **`gcal_list_events`** — その日の全イベントを取得（前の予定の終了時刻を正確に把握するため）
2. **`gcal_find_my_free_time`** — 空きスロットを取得（`minDuration` = 枠の長さ）

候補数が満たされるまで日付を進める。一度に取得しすぎず、対象曜日だけを効率よく問い合わせる。

### Step 3: 空き枠の計算

各日のイベント情報と空きスロット情報を組み合わせて、以下のルールで候補枠を算出する。

#### バッファの適用
- 空きスロットの開始時刻が、直前のイベント終了時刻＋バッファ未満の場合、開始時刻を「直前のイベント終了＋バッファ」に繰り上げる
- 繰り上げた結果、残りの空き時間が枠の長さ未満になったら、そのスロットは候補から除外する
- **開始時刻は15分単位（:00, :15, :30, :45）に切り上げる**（例: 11:47 → 12:00）

#### 連続空きのマージ
- 1時間枠が複数連続して確保できる場合、1つのブロックにまとめる
  - 例: 13:00-14:00 と 14:00-15:00 が両方空き → `13:00-15:00` と表記
- これにより、相手が好きな時間を選びやすくなる

#### 除外条件
- 終日イベント（allDay: true）は、`workingLocation` や `outOfOffice` の `transparency: "transparent"` でない限りブロックとみなす
- ステータスが `declined` のイベントはブロックしない

### Step 4: 出力フォーマット

以下の形式でコピペしやすく出力する。

```
・{M/D} ({曜日}) {開始}-{終了}
・{M/D} ({曜日}) {開始}-{終了}, {開始}-{終了}
```

**フォーマットルール:**
- 日付は `M/D (曜日)` 形式（例: `4/13 (月)`）
- 時刻は `HH:MM` の24時間表記
- 同じ日に複数の空きブロックがある場合はカンマ区切りで1行にまとめる
- 曜日は日本語1文字（月火水木金土日）

**出力例:**
```
・4/6 (月) 12:00-14:00, 15:00-19:00
・4/8 (水) 12:00-14:00, 16:15-19:00
・4/13 (月) 12:00-19:00
・4/15 (水) 12:00-19:00
・4/20 (月) 12:00-19:00
```

### Step 5: 補足説明

候補リストの後に、簡潔に以下を添える:
- 各日の主な既存予定（なぜその時間帯が空いているかの文脈）
- 共通パターンがあれば一言で（例:「毎日11:00-11:45にデイリーがあるため最速12:00開始」）

## 注意事項

- **効率的なAPI呼び出し**: 候補数が十分集まったら、それ以上の日付は問い合わせない
- **ユーザーが曜日を指定した場合**: その曜日だけを問い合わせる（全日取得しない）
- **祝日の考慮**: 祝日判定はAPIで行えないため、ユーザーに確認が必要な場合は聞く
- **出力は即コピペ可能**: 余計な装飾やMarkdown記法は使わない。先方に送るテキストとしてそのまま使える形にする
